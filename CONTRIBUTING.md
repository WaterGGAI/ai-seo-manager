# Contributing

Contributions are welcome.

## Ways To Contribute

- Open issues for bugs or UX problems
- Propose connector patterns for new website types
- Improve deployment examples
- Improve docs and onboarding flows
- Submit PRs for isolated modules or fixes

## Project Principles

- Keep the architecture practical for small teams
- Prefer Cloudflare-native patterns where they simplify operations
- Keep multi-site support a first-class concern
- Document tradeoffs clearly
- Avoid locking the system into one content model or one website type

## Before Opening A PR

- Explain the problem being solved
- Keep changes scoped
- Add or update docs when behavior changes
- Avoid introducing secrets or account-specific values into the repository
